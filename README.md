### 복합 인덱스 컬럼 순서 검증

친절한 SQL 튜닝에서 배운 두 가지 명제를 직접 검증했다.

1. **= 조건 컬럼끼리는 인덱스 순서를 바꿔도 비용이 같다**
2. **범위 조건은 인덱스 맨 뒤에 두어야 한다**

#### 검증 환경
- Oracle XE 21c (Docker)
- 테스트 데이터: 100만 건 (trade_history)

#### 검증 결과

| 인덱스 | 컬럼 구성 | 스캔 방식 | 인덱스 Buffers | 총 Buffers |
|--------|-----------|-----------|----------------|------------|
| IDX_A | (등급, 번호, 일자) | INDEX RANGE SCAN | 3 | 27 |
| IDX_B | (번호, 등급, 일자) | INDEX RANGE SCAN | **3** | **27** |
| IDX_C | (등급, **일자**, 번호) | INDEX SKIP SCAN | **83** | **107** |

#### 결론
- **IDX_A == IDX_B** : `=` 조건 컬럼은 순서를 바꿔도 비용이 동일하다 (Buffers 3으로 같음)
- **IDX_C** : 범위 조건(일자)을 가운데 두면 뒤 컬럼(번호)이 filter로 전락하고
  옵티마이저가 RANGE SCAN을 포기하고 SKIP SCAN으로 우회한다 → **I/O가 약 28배 증가**
